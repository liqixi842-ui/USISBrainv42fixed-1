USIS Brain v3.1 · MVP 智能体（今日可上线）
0) 范畴与目标

只改大脑（Replit/Node），n8n 已稳定作为“发送器官”，无需改动。

上线即用的三项能力

解析全球任意国家股票/指数/ETF/期货/外汇/加密资产（简称“标的解析”）。

返回近2小时内可能影响股市的新闻热点，按紧急度排序。

用户可点播资讯/分析/建议三种输出类型（可单独要，也可组合要）。

1) 顶层意图与调用方式
1.1 输入（统一入参）
{
  "text": "西班牙热力图 + FER 解析 + 两小时内影响市场的新闻",
  "user_id": "tg:75613...",
  "chat_type": "private",
  "locale": "zh-CN",           // 用户语言
  "region_hint": "ES",         // 可为空；影响歧义解析
  "mode": "auto",              // auto/news/analysis/advice（可覆盖）
  "budget": "medium",          // low/medium/high/unlimited
  "time_horizon": "2h"         // 仅新闻收集时使用，默认2h
}

1.2 意图分类（MVP）

PARSE（标的解析）

NEWS（两小时热点新闻）

ANALYZE（分析）

ADVISE（建议）

一次请求可包含多个意图；解析优先，其结果供后续 NEWS/ANALYZE/ADVISE 使用。

2) 数据源策略（MVP 级别）

全市场标的解析：Finnhub symbol lookup + exchange map（主力：US/EU/ES/JP/HK）；本地缓存常见映射（如 IBEX/ES、DAX/DE、CAC/FR、FTSE/UK、TSE/JP、HKEX/HK、NASDAQ/NYSE/AMEX/US、SIX/CH、ASX/AU 等）。

实时/延时价格：Finnhub quote（或你现有的报价节点）。

宏观：FRED（CPI、失业率、GDP、利率等）。

公司披露：SEC EDGAR（US）；ES/DE/FR 如无官方 API，先不强求，MVP 用财经媒体新闻替代。

新闻：优先使用现有已通的源（你已有可用源就用它）；若多源并存，按以下顺序拉取：

公司级新闻：与解析出的 symbols 绑定；

市场级新闻：与 region_hint/指数相关（ES: IBEX、US: SPX/NDX 等）；

宏观/政策：央行、就业、通胀、地缘事件。

今天上线：若某地区新闻源不足，使用国际综合源+关键词过滤（含指数/板块/龙头词），保证“2小时”窗口内有内容且能排序。

3) 核心决策与排序
3.1 标的解析（全局可用）

规则：显式前缀 > 主流映射 > 地域/语言加权 > Top3 候选

输出：

{
  "symbols": [
    {"input":"FER","resolved":"ES:FER","name":"Ferrovial SE","type":"equity","exchange":"BME"},
    {"resolved_index":"ES:IBEX35"}
  ],
  "is_adr_pair":[/* 如遇台积电/TSMC ADR 同名时 */],
  "disambiguation": false
}

3.2 两小时热点新闻排序（ImpactRank v0）

打分 = 主题相关度 × 来源权威度 × 新鲜度衰减 × 市值权重

主题相关度：与 symbols/sector/region 的匹配（TF-IDF/关键词命中）。

来源权威度：白名单>主流媒体>次级媒体。

新鲜度衰减：exp(-Δt/40min)；Δt≤120min。

市值权重：涉及大市值/权重股/指数的，+权重。

产出（Top N，默认 5 条）：每条附 impact_score、reason（为什么重要）、time、source、tickers[]。

3.3 三分离输出策略

资讯（NEWS_ONLY）：只列要点 + 来源 + 影响简评（1句）。

分析（ANALYSIS_ONLY）：结构化推理（背景→机制→市场链路→最可能场景）。

建议（ADVICE_ONLY）：仓位/风险/观察点/触发条件（不提供个股买卖信号，仅给方法论与风控位）。

组合：按用户 mode 合并段落（顺序：资讯→分析→建议）。

5) 模型选择与预算（MVP 版）

low：轻模型（GPT-mini/Haiku）＋零深推

medium（默认）：轻模型为主，分析段落允许调用一次中档模型

high：中档为主，场景推演/建议可启一次深推

unlimited：允许 L3（o1/Opus）用于复杂多标的组合分析

今天先按规则生效；后续再把 token 成本写入 Postgres 的 cost_tracking。

6) 失败与兜底

解析失败：返回 disambiguation=true 与 Top3 候选；仍可给“市场级”新闻与宏观分析。

新闻缺失：自动放宽到 6 小时 + 提醒“2小时内无显著事件”。

模型超时/报错：降级为 NEWS_ONLY + 简短基线分析（模板化）。

图表失败：仅发文字；不阻断信息流。

组合输出

文本：AAPL 解析；给我新闻+建议

期望：解析+AAPL相关两小时新闻+仓位与风控建议；如需要，触发 index_pair(NDX,SPX) 图。
